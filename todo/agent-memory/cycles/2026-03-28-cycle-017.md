## Goal

Implement e2e server lifecycle auto-detect so BFF e2e reuses already-running servers and auto-starts when absent.

## Plan

1. Run pre-chunk validation.
2. Add server probe/start/teardown guards to consumer and admin BFF e2e support files.
3. Remove forced Nx `dependsOn` startup for BFF e2e targets.
4. Run post-chunk validation and targeted e2e commands.
5. Sync memory files.

## Actions

- Ran `npm run format && npx nx affected -t lint test typecheck --uncommitted --tui=false` before edits.
- Updated BFF e2e setup/teardown files to:
  - probe target port,
  - reuse running server when available,
  - spawn `nx run <bff>:serve` when unavailable,
  - stop server only when started by e2e.
- Removed BFF e2e `dependsOn` startup entries from both e2e `package.json` files.
- Ran post-checks and targeted e2e specs for consumer/admin BFF scopes.

## Results

- BFF e2e now follows desired runtime behavior: reuse-if-running, start-if-not-running.
- Teardown no longer blindly kills user-started servers when reuse path is detected.
- Verification passed for affected lint/test/typecheck and both targeted BFF e2e runs.

## Learnings

- Playwright has native reuse/start behavior via `webServer.reuseExistingServer`; Jest e2e requires explicit setup/teardown lifecycle code.
