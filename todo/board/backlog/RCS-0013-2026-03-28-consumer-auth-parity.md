# RCS-0013 consumer-auth-parity

## Code

- `RCS-0013`

## Goal

Mirror proven admin auth/session patterns to consumer apps with consumer-specific route behavior.

## Acceptance

- [ ] Consumer auth/session endpoints or handlers follow same contract style.
- [ ] Consumer web handles authenticated and unauthenticated states.
- [ ] Tests confirm parity for success and deny paths.

## Notes

Reuse patterns, not cross-surface coupling.
